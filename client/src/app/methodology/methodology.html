How Solar Score Is Calculated

Solar Harvest — scoring methodology & data sources

star Solar Score (0 – 100) A weighted composite of three factors

Each nonprofit building receives a Solar Score between 0 and 100. A higher score means a solar installation is likely to generate more energy and have a larger financial impact relative to the organization's size.

50%
Solar Yield
How much electricity the roof can generate per year (kWh/yr ÷ 500,000 cap). A 100 kW system in Kentucky produces roughly 120,000–145,000 kWh/yr.
35%
Financial Impact
Estimated annual savings as a percentage of 10% of the org's reported revenue. Smaller nonprofits score higher here — a $15,000 savings means more to a $200k/yr food bank than to a $50M hospital.
15%
Roof Coverage
Fraction of the roof usable for panels (from Google Project Sunroof when available; defaults to 50% neutral when roof geometry data is unavailable from PVWatts).

 savings Estimated Annual Savings Based on projected electricity output and a flat commercial rate

Savings are estimated using a per-location electricity rate:

Est. Savings = Annual kWh Output × Local Rate ($/kWh)

The electricity rate is looked up per building using the NREL Utility Rate Database (URDB) — a free government database of utility rate structures for every US zip code. When URDB returns a usable flat rate, that rate is used. Otherwise the system falls back to the EIA state commercial average (e.g., Kentucky: $0.077/kWh, 2024). The rate actually used for each building is shown in the building detail panel.

The figure does not include installation costs, incentives (federal ITC, state rebates), or net-metering credits — it is a gross energy-offset estimate only.

solar_power Solar Modeling Assumptions NREL PVWatts v8 — default commercial system

Solar output is modeled using the NREL PVWatts API v8, a free government tool that estimates annual AC electricity production for a photovoltaic system at any US location.

Building-Specific System Sizing

Rather than assuming a fixed system size, Solar Harvest queries OpenStreetMap for each building's footprint polygon. The usable roof area is calculated from the polygon using the Shoelace formula, then converted to a system capacity:

System kW = Footprint m² × 75% usable × 0.150 kW/m²

If no OSM building footprint is found within 60 m of the geocoded address, the model falls back to a 100 kW default. The derived system size and roof area are shown in the building detail panel.

source Data Sources
IRS Exempt Organizations BMF
Primary list of ~1.95 million 501(c) organizations. Provides name, EIN, address, NTEE category, assets, and revenue. Updated monthly; Solar Harvest checks daily for new releases.
IRS Form 990 XML (ProPublica / IRS S3)
E-filed 990 returns contain narrative descriptions of programs and facilities. Solar Harvest parses these to discover additional building addresses (Schedule O, Part III narratives) beyond the primary IRS mailing address.
US Census Geocoder
Free geocoding API (geocoding.geo.census.gov) used to convert street addresses to latitude/longitude. No API key required. Google Maps Geocoding is used as a fallback when a Census key is configured.
NREL PVWatts API v8
National Renewable Energy Laboratory solar irradiance and production model. Free API (developer.nrel.gov) with up to 1,000 requests/hour. Returns annual AC energy output for a defined system at any US coordinate.
NREL Utility Rate Database (URDB)
Free database of commercial electricity rate structures for every US zip code. Provides actual utility rates ($/kWh) per location rather than a flat national average. Falls back to EIA state averages when a simple flat rate cannot be extracted from the utility's rate structure.
OpenStreetMap (Overpass API)
Free, community-maintained building footprint data. Used to determine the actual roof area of each nonprofit building, enabling per-building system sizing rather than a uniform 100 kW assumption.
