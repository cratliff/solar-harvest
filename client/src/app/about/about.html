About Solar Harvest

Data sources, scoring methodology, and project overview

What is Solar Harvest?

Solar Harvest is a free, open-source tool that ranks US nonprofits by their potential to benefit from rooftop solar installations. It combines IRS financial data, government solar irradiance models, and building footprint data to produce a ranked list of the organizations where solar would generate the most electricity and have the greatest financial impact.

The tool is designed for solar installers prospecting for commercial customers, clean-energy advocates seeking grant targets, and nonprofits themselves evaluating whether solar makes economic sense for their facilities.

1 — IRS Data Import
The IRS publishes a monthly Business Master File (BMF) listing every 501(c) organization in the country — roughly 1.95 million records. Solar Harvest downloads and imports this file, capturing each org's name, EIN, primary mailing address, NTEE sector code, total assets, and annual revenue. IRS e-filed Form 990 XML returns are also parsed to discover additional building addresses mentioned in Schedule O narratives.
place
2 — Geocoding
Each address is converted to a latitude/longitude coordinate using the US Census Geocoder (free, no API key). Addresses that fail Census geocoding fall back to Google Maps Geocoding if a key is configured. Only geocoded addresses can receive a solar score.
roofing
3 — Building Sizing via OpenStreetMap
For each geocoded address, Solar Harvest queries the OpenStreetMap Overpass API for building footprint polygons within 60 m. The polygon area is computed via the Shoelace formula and projected to square meters. System capacity is derived as:
System kW = Footprint m² × 75% usable × 0.150 kW/m²
The 75% factor accounts for HVAC equipment, roof edges, and shading. If no footprint is found, a 100 kW default is used. System capacity is clamped between 5 kW and 1,000 kW to guard against polygon errors.
solar_power
4 — Solar Output via NREL PVWatts
Annual AC electricity production is estimated using the NREL PVWatts v8 API — a free government model that computes solar irradiance and system output for any US coordinate. To stay within the 1,000 req/hour free tier, Solar Harvest pre-populates a geographic grid of unit-capacity (1 kW) results at 0.1° resolution for each state. Building-level output is then a simple local lookup:
Annual kWh = Grid cell kWh/kW × System kW
This means a single PVWatts call covers roughly every building within ~7 km, making state-wide scoring practical within the free quota.
star
5 — Scoring & Rate Lookup
Each building receives a Solar Score (0–100) and an estimated annual savings figure. Electricity rates come from the NREL Utility Rate Database (URDB) — a per-zip-code commercial rate lookup — pre-cached in MongoDB. State EIA averages are used as a fallback when URDB returns no usable flat rate.
star The Solar Score (0 – 100) A weighted composite of three factors

Each building receives a Solar Score between 0 and 100. A higher score means a solar installation would generate more energy and have a larger financial impact relative to the organization's size.

50%
Solar Yield
How much electricity the roof can generate per year (kWh/yr ÷ 500,000 cap). A 100 kW system in Kentucky produces roughly 120,000–145,000 kWh/yr.
35%
Financial Impact
Estimated annual savings as a share of 10% of reported revenue. Smaller orgs score higher — a $15,000 savings matters more to a $200K food bank than to a $50M hospital.
15%
Roof Coverage
Fraction of the roof usable for panels. Defaults to 50% when roof geometry data is unavailable (PVWatts-only path).

Estimated Annual Savings

Savings are calculated per building as:

Est. Savings = Annual kWh × Local Rate ($/kWh)

The figure represents gross energy-offset value only. It does not include installation costs, federal Investment Tax Credit (ITC), state rebates, SREC revenue, or net-metering credits.

source Data Sources All data is sourced from free, public government and open-data APIs
business
IRS Exempt Organizations Business Master File (BMF)
Primary list of ~1.95 million 501(c) organizations published monthly by the IRS. Provides name, EIN, mailing address, NTEE sector code, total assets, and annual revenue. Solar Harvest checks daily for new releases and re-imports automatically.
irs.gov · Updated monthly · No API key required
description
IRS Form 990 XML (via IRS S3 / ProPublica)
E-filed 990 returns contain narrative descriptions of programs and facilities. Solar Harvest parses Schedule O and Part III narratives to discover additional building addresses beyond the primary IRS mailing address, giving a more complete picture of each org's real-estate footprint.
irs.gov S3 · Annual filings · No API key required
location_on
US Census Bureau Geocoder
Free REST API (geocoding.geo.census.gov) that converts street addresses to latitude/longitude coordinates using the Census TIGER/Line database. No API key is required. Google Maps Geocoding is used as a fallback when a key is configured.
census.gov · Free · No API key required
solar_power
NREL PVWatts API v8
National Renewable Energy Laboratory solar irradiance and production model. Returns annual AC electricity output (kWh) for a photovoltaic system at any US coordinate, accounting for local weather, panel tilt, azimuth, and system losses. Free tier: up to 1,000 requests/hour. Solar Harvest pre-computes a 0.1° geographic grid per state so that building-level scoring consumes zero live API quota during normal operation.
developer.nrel.gov · Free · NREL_API_KEY required
electric_bolt
NREL Utility Rate Database (URDB)
Free government database of commercial electricity rate structures for every US zip code. Provides actual utility rates ($/kWh) per location rather than a flat national average, enabling more accurate savings estimates. URDB results are cached in MongoDB (90-day TTL). When URDB returns a complex time-of-use structure without a clear flat rate, the system falls back to EIA 2024 state commercial averages.
openei.org · Free · NREL_API_KEY required
roofing
OpenStreetMap (Overpass API)
Free, community-maintained building footprint dataset. Used to determine the actual roof area of each nonprofit building via polygon geometry (Shoelace formula, equirectangular projection). Enables per-building system sizing rather than a uniform 100 kW assumption. Coverage varies by region; rural buildings may not have footprint data.
openstreetmap.org · Free · No API key required
 warning_amber Limitations & Caveats What this tool does not account for
attach_money
Installation costs are not modeled. The savings estimate is gross energy-offset value only. Actual ROI depends on installed system cost (typically $1.50–$2.50/W commercial), available incentives (federal 30% ITC, state rebates), and financing terms.
grid_off
Net metering and grid export are not included. In states with strong net-metering policies, actual financial value may be higher. Demand-charge reduction — often the largest savings driver for commercial customers — is also not modeled.
park
Tree shading and local obstructions are not analyzed. PVWatts uses a climate-based irradiance model, not a site-specific shading analysis. Actual production can be meaningfully lower for buildings surrounded by tall trees or adjacent structures.
engineering
Structural suitability is not assessed. Roof age, load capacity, and membrane condition determine whether a building can physically support panels. A high Solar Score does not indicate a roof is ready for installation.
map
OSM building footprint coverage is incomplete in rural areas. Buildings without an OSM footprint default to a 100 kW system, which may over- or under-estimate the true roof area. Urban and suburban areas generally have much better coverage.
business_center
IRS mailing addresses may not reflect facility locations. Many nonprofits file taxes from an accountant's or attorney's office. The 990 narrative parsing improves this, but some organizations may have buildings not captured in any filing.
